# AgentPulse 实现计划与代码结构

本文档是 AgentPulse 的实现索引。新增、移动、重命名或删除业务目录时，必须同步更新本文档，使其始终与真实业务代码目录结构一致。

每个业务目录都必须说明职责、主要入口文件、关键公共类型或接口、数据流和测试要求。具体模块的内部设计见[模块详细设计](../03-模块详细设计/README.md)。

## 根目录配置

- 职责：工作区编排、共享 TypeScript 配置、包脚本和文档结构检查。
- 主要文件：`package.json`、`pnpm-workspace.yaml`、`tsconfig.base.json`、`turbo.json`、`AGENTS.md`、`.github/workflows/*.yml`、`scripts/check-doc-structure.mjs`、`scripts/check-release-version.mjs`。
- 公共接口：根脚本 `build`、`test`、`dev`、`start`、`agent-pulse`、`check:docs`，以及 npm 包发布辅助脚本。
- 数据流：根脚本通过 pnpm 过滤器和 Turbo 把任务分派到各应用与包；GitHub Actions 对所有分支推送和拉取请求执行持续集成。只有版本匹配 `vX.Y.Z` 且对应提交已包含在 `origin/main` 中的标签才能发布 npm 包。
- 测试要求：根目录的构建、测试和检查脚本必须覆盖所有业务包，并校验实现计划的文档结构。

## apps/cli

- 职责：提供 `agent-pulse` 命令和可发布给用户的 npm 包。
- 主要文件：`src/index.ts`、`package.json`。
- 公共接口：`scan`、`inventory`、`plan`、`install`、`rollback`、`start`、`doctor`、`--version` 命令，以及 npm 可执行入口 `agent-pulse`。
- 数据流：命令行程序调用能力盘点、安装器、存储和服务端模块。`start` 在监听端口前启动 Fastify 应用，并解析已打包的 Web 与文档静态资源。
- 测试要求：版本与帮助输出、只读的扫描/盘点/计划行为、备份/应用/回滚语义，以及打包后的 `agent-pulse start` 能够提供控制台和文档服务。

## apps/server

- 职责：提供本地 Fastify 服务、HTTP API、代理路由、Web 控制台的服务端渲染与静态资源服务，以及文档站点的静态资源服务。
- 主要文件：`src/index.ts`、`src/app.ts`。
- 公共接口：`/api/health`、Hook 事件接收、事件/任务/会话、能力盘点、安装计划、Agent 扫描/替换/回滚、代理捕获列表/详情/会话摘要、通知测试、`/docs/` 静态文档，以及控制台的服务端渲染回退入口。
- 数据流：API 请求写入存储，分析器标记风险，通知渠道发送通知，代理路由在不缓冲流式响应的前提下转发流量，静态请求读取构建产物。
- 测试要求：健康检查、Hook 幂等、能力盘点扫描、Agent 计划/备份/应用/回滚、包含流式兼容性的代理透传、代理请求详情查询、文档静态路由和服务端渲染回退。

## apps/web

- 职责：提供本地运行管理控制台。
- 主要文件：`src/entry-client.tsx`、`src/entry-server.tsx`、`src/Root.tsx`、`src/App.tsx`、`src/styles.css`。
- 公共接口：面向用户的 Agent、会话、事件、任务、能力盘点、计划、代理请求列表/详情、通知和诊断页面，以及指向 `/docs/` 的文档链接。
- 数据流：浏览器只访问 `/api/*`，不直接读取本地文件。会话页面先读取代理会话摘要，再按选中的 `sessionId` 过滤代理请求捕获记录。代理详情视图展示经过脱敏的请求体与响应内容，不在界面展示捕获的请求头。服务端渲染初始页面骨架，客户端通过 TanStack Query 完成水合。
- 测试要求：Vite 客户端构建、Vite 服务端渲染构建、主要页面渲染、带代理会话选择的会话页面、代理详情抽屉、空状态和文档链接。

## apps/docs

- 职责：提供基于 VitePress 的使用文档，覆盖安装、启动、Web 控制台流程、命令行命令、代理配置和安全回滚。
- 主要文件：`index.md`、`guide/*.md`、`.vitepress/config.ts`、`package.json`。
- 公共接口：由 `agent-pulse start` 提供的 `/docs/` 静态站点，以及包内 `dev` 脚本启动的本地文档开发服务。
- 数据流：Markdown 文件构建到 `.vitepress/dist`；服务端把该目录作为静态资源提供，不读写用户的本地配置。
- 测试要求：VitePress 构建成功，主要指南页面的链接可正常解析。

## packages/core

- 职责：提供标准类型、状态模型、标识符、时间戳和脱敏工具。
- 主要文件：`src/index.ts`。
- 公共接口：`AgentEvent`、`TaskStatus`、`InstallPlan`、能力盘点条目类型、探针结果类型、`newId`、`nowIso`、`redactSecrets`、`stableId`。
- 数据流：由服务端、存储、能力盘点、集成、安装器、代理和分析器共享。
- 测试要求：事件标准化辅助函数、标识符生成、任务状态推导和敏感信息脱敏。此包中禁止出现工具特定逻辑。

## packages/storage

- 职责：使用纯 JavaScript/WASM 依赖提供本地优先、兼容 SQLite 的存储和结构初始化，确保 npm 安装时不需要编译原生 SQLite。
- 主要文件：`src/index.ts`。
- 公共接口：`createStorage`、`AgentPulseStorage`，以及事件、任务、会话、代理列表/详情/会话摘要、能力盘点、安装计划、备份和设置相关方法。
- 数据流：命令行程序、服务端、安装器、能力盘点和代理共享同一套存储接口。代理捕获记录可以在持久化前携带由集成适配器提取并标准化的 `sessionId`。敏感信息必须在持久化前脱敏。
- 测试要求：结构初始化与增量迁移、事件幂等写入、备份、能力盘点更新或插入、包含会话标识的代理详情查询、代理会话摘要、路由映射，以及最近已应用备份的查询。

## packages/probes

- 职责：提供只读的本地事实探针。
- 主要文件：`src/index.ts`。
- 公共接口：文件系统、配置、技能、MCP、敏感信息和进程探针辅助函数。
- 数据流：能力盘点、集成和安装器通过探针收集事实，避免重复实现环境判断。
- 测试要求：技能发现、MCP 配置提取、敏感字段脱敏和命令存在性检查。

## packages/inventory

- 职责：扫描本地技能、MCP 服务器、插件和配置来源。
- 主要文件：`src/index.ts`。
- 公共接口：`scanInventory`、`diffInventory`。
- 数据流：从集成适配器获取配置来源声明，通过探针读取元数据，再把清理后的结果写入存储。
- 测试要求：工作区级、用户级和全局级来源发现，缺失路径警告，以及不存储任何敏感信息明文。

## packages/integrations

- 职责：提供工具集成适配器和共享的适配器辅助能力。
- 主要文件：`src/index.ts`、`src/types.ts`、`src/adapters/*`、`src/helpers/*`。
- 公共接口：适配器注册表、扫描/计划/验证辅助函数和代理路由配置。
- 数据流：每个适配器独立负责对应工具的配置发现和补丁逻辑；安装器消费标准安装计划和代理配置。
- 测试要求：Codex TOML 补丁、Claude JSON/JSONC 补丁、OpenCode 提供方补丁、注册表行为、路由状态验证和命令建议降级。

## packages/installer

- 职责：扫描、计划、备份、应用、验证和回滚配置变更。
- 主要文件：`src/index.ts`。
- 公共接口：`scan`、`planInstall`、`applyInstall`、`rollbackLatest`、`rollbackIntegration`。
- 数据流：配置变更必须始终遵循“计划 -> 备份 -> 应用 -> 验证”；回滚根据备份元数据恢复或删除文件。
- 测试要求：计划阶段无副作用、备份元数据、应用后验证、回滚恢复或删除，以及按集成范围回滚。

## packages/proxy

- 职责：提供 `/proxy/{proxyKey}/*` 下的本地代理。
- 主要文件：`src/index.ts`、`src/serializers.ts`、`src/parsers.ts`。
- 公共接口：`registerProxyRoutes`。
- 数据流：请求先从存储中解析路由映射，保留请求方法、路径、查询参数、请求头和请求体，再根据 API 协议及模型或提供方提示选择请求序列化器；随后提取 Claude Code 会话标识、提示词片段等 Agent 特定元数据并转发到上游。面向用户的响应直接流式透传；响应解析器按 Agent 和 `主版本.次版本` 选择，最后把脱敏后的请求与响应详情写入存储，供 Web 控制台查看上下文。

  序列化器父类覆盖 OpenAI 兼容和 Anthropic 兼容的默认行为。GPT/ChatGPT、DeepSeek、GLM 等模型公司的子类只在确有特殊序列化需求时覆盖默认行为，否则继承父类。解析器按 Agent 区分并感知版本；找不到完全匹配的 `主版本.次版本` 时回退到最新版本。未来可以在解析器注册表边界增加动态解析器加载，不需要修改代理路由处理器。

  Claude Code 请求元数据解析会读取 `body.metadata.user_id.session_id`，也兼容 `user_id` 本身是 JSON 字符串的情况，并把结果标准化为代理请求记录上的 `sessionId`，同时保留脱敏后的请求体捕获。提示词解析会把请求体内容归纳为 `promptParts`，涵盖系统、用户、助手、工具调用/结果、MCP 调用/结果和技能引用；原始请求体捕获仍是事实来源。

  请求捕获使用实际转发的请求体，保存完整的脱敏请求体，不应用界面展示长度限制。捕获的请求头与响应体分别设置大小上限，避免过大的元数据遮蔽界面中的请求体或响应内容。非流式文本或 JSON 响应保存脱敏后的响应体详情。SSE 响应保持面向用户的低开销直通链路，只在流式传输期间缓冲有界的原始响应字节，并在流结束后解析和写入结构化的模型、用量、思考内容、文本和工具调用摘要。用量中的令牌数量需要保留，可能包含敏感信息的令牌字段仍需脱敏。二进制响应只捕获元数据。
- 测试要求：路由映射命中、未知键返回 404、协议/模型序列化器选择与回退、从对象和 JSON 字符串形式的 `metadata.user_id` 提取 Claude Code 会话标识、提取系统/用户/助手/工具/MCP/技能类别的提示词片段、按 Agent `主版本.次版本` 选择解析器并回退到最新版本、非流式请求/响应详情捕获、原始请求体捕获、超大输入的完整脱敏请求体捕获、SSE/分块响应直通以及流结束后的结构化模型/用量/思考内容/文本/工具调用提取、上游 4xx/5xx 透传、本地 502 处理、敏感请求头/请求体脱敏，以及不会保存敏感信息明文的响应和请求头截断。

## packages/channels

- 职责：提供通知渠道。
- 主要文件：`src/index.ts`。
- 公共接口：通过 `sendNotification` 提供 Webhook 和 Windows 通知发送能力。
- 数据流：服务端调用通知渠道发送消息，并把发送结果写入存储。
- 测试要求：Webhook 请求体、超时与错误行为，以及 Windows 通知降级。

## packages/analyzers

- 职责：提供基础事件和请求分析。
- 主要文件：`src/index.ts`。
- 公共接口：`analyzeEvent`。
- 数据流：服务端接收事件或捕获代理请求时调用分析器，并保存分析结果和风险等级。
- 测试要求：API 密钥模式识别、危险命令识别、普通低风险事件和失败分类。

## plugins

- 职责：作为最小可行产品阶段的示例插件内容占位目录；运行时不加载不受信任的动态插件。
- 主要文件：存在插件时使用对应插件目录。
- 公共接口：当前运行时不提供公共接口。
- 测试要求：除非明确引入插件运行时加载，否则只检查目录存在性。
